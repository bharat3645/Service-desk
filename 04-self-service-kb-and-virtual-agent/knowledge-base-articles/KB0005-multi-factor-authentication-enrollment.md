# Multi-Factor Authentication Enrollment

## Prerequisites

An authenticator application installed on a mobile device (Microsoft
Authenticator or Google Authenticator — confirm the organization
standard before proceeding).

## Resolution steps

1. Navigate to the MFA enrollment page (linked from the Self-Service
   Portal home screen under **"Security Settings"**).
2. Authenticate with existing username and password.
3. Select **"Add a method"** → **Authenticator App**.
4. In the authenticator application, select **"+"** → **"Scan QR
   code"**, then scan the code displayed on screen.
5. Enter the 6-digit code generated by the application to confirm
   pairing.
6. Optionally configure a backup method (SMS to a personal number) for
   device-loss contingency.

## Troubleshooting

| Symptom | Cause | Resolution |
|---|---|---|
| QR code will not scan | Camera permission or lighting issue | Use "Enter code manually" and type the key shown beneath the QR code |
| Generated code rejected as invalid | Device clock out of sync | Enable automatic date and time synchronization on the mobile device |
| Device lost or replaced | No backup method configured | Re-enroll via backup method if configured; otherwise requires Service Desk identity verification |

## When to contact the Service Desk instead

Escalate immediately if the sole MFA-enrolled device is lost with no
backup method configured — this scenario requires identity verification
before MFA enrollment can be reset.
